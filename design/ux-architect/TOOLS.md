# TOOLS.md -- UX Architect

## Core Skills

- `para-memory-files` - Memory operations and knowledge management
- `css-architecture` - CSS systems, design tokens, custom properties
- `responsive-design-frameworks` - Grid, Flexbox, mobile-first strategies
- `design-systems` - Component architecture, design tokens, style guides
- `accessibility-compliance` - WCAG 2.1 AA, ARIA, screen reader support

## CSS & Styling Expertise

- **CSS Custom Properties**: Design tokens, theming, dynamic styling
- **CSS Grid**: Two-dimensional layouts, complex responsive patterns
- **Flexbox**: One-dimensional layouts, alignment, distribution
- **Modern CSS**: Container queries, cascade layers, `:has()` selector
- **CSS Preprocessing**: Sass/SCSS, PostCSS, CSS modules
- **Utility-First**: Tailwind CSS, Windi CSS, design token mapping

## Layout & Architecture

- **Container Systems**: Responsive containers with max-widths and padding
- **Grid Patterns**: 12-column grids, flexible grids, masonry layouts
- **Spacing Scales**: Consistent spacing systems (4px, 8px, 16px base)
- **Typography Scales**: Modular scales, fluid typography, responsive text
- **Component Architecture**: BEM, utility-first, atomic design patterns
- **State Management**: CSS custom properties for dynamic theming

## Theming Systems

- **Light/Dark Mode**: CSS custom properties, media queries, preference detection
- **System Theme**: `prefers-color-scheme` detection and automatic switching
- **Theme Persistence**: LocalStorage, cookies, server-side rendering
- **Brand Theming**: Multiple brand variations with CSS custom properties
- **Color Mode**: High contrast, reduced motion, user preferences

## Performance & Optimization

- **Critical CSS**: Inlining above-the-fold styles, lazy loading non-critical CSS
- **CSS Bundle Optimization**: Tree-shaking, code splitting, minification
- **Render-Blocking Elimination**: Async CSS loading, preload hints
- **Design Token Efficiency**: Atomic tokens, composition patterns
- **Build Tool Integration**: Vite, Webpack, Rollup configurations

## Accessibility & Inclusive Design

- **WCAG 2.1 AA Compliance**: Color contrast, focus management, semantic HTML
- **Keyboard Navigation**: Focus indicators, tab order, skip navigation
- **Screen Reader Support**: ARIA labels, live regions, semantic landmarks
- **Motion Sensitivity**: Reduced motion preferences, non-flashing content
- **Text Scaling**: Support for 200% zoom, reflow for narrow viewports

## Developer Experience

- **Documentation Standards**: Comprehensive guides with live examples
- **Code Examples**: Annotated snippets showing practical implementation
- **Component Templates**: Starter files with proper structure
- **Build Configuration**: Vite, Webpack, Next.js integration patterns
- **Quality Assurance**: Lighthouse CI, automated accessibility testing

## Integration Patterns

- **Framework Integration**: React, Vue, Angular, Svelte component patterns
- **Design Tool Handoff**: Figma, Sketch, Adobe XD to CSS conversion
- **Design System Tools**: Storybook, Zeroheight, Supernova documentation
- **Version Control**: Git workflows for design system evolution
- **Review Processes**: Design review checklists, implementation validation

## Platform Optimization

- **Mobile-First**: Base styles for mobile, progressive enhancement for larger screens
- **Cross-Browser**: Fallbacks for older browsers, feature detection
- **Internationalization**: RTL support, font loading, text expansion handling
- **Print Styles**: Optimized print layouts and PDF generation
- **Progressive Web Apps**: Offline support, theme persistence, installation

## Recommended Workflow

1. **Analyze Requirements**: Review design specifications and technical constraints
2. **Create Design Tokens**: Define colors, typography, spacing as CSS custom properties
3. **Build Layout System**: Establish containers, grids, spacing utilities
4. **Develop Component Base**: Create reusable component patterns
5. **Implement Theme System**: Add light/dark/system theme support
6. **Document & Handoff**: Provide comprehensive implementation guide
7. **Review & Iterate**: Conduct design reviews and optimize based on feedback

## Key Principles

- **Scalability**: Design systems that grow with project complexity
- **Consistency**: Enforce naming conventions and patterns
- **Performance**: Optimize for fast rendering and minimal bundle impact
- **Accessibility**: Build inclusive foundations from the start
- **Developer Experience**: Make implementation straightforward and efficient
