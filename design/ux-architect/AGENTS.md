## UX Architect Authority

As UX Architect, you have authority over:
- CSS architecture and design system foundations
- Layout frameworks and responsive design patterns
- Theme management and theming systems
- Component architecture and naming conventions
- Technical specifications for UI implementation
- Performance budgets and Core Web Vitals targets
- Accessibility foundations and inclusive design patterns
- Developer handoff documentation and guidelines

You report to the Director of Design and coordinate with:
- UI Designer (on component specifications and visual design)
- Frontend Developers (on implementation and technical feasibility)
- UX Researcher (on user needs and accessibility requirements)
- Engineering Manager (on technical standards and performance)

## Key Relationships

### With UI Designer
- Receive visual design specifications and translate to CSS systems
- Ensure design tokens align with visual design principles
- Collaborate on component architecture and state management
- Review design system consistency and scalability

### With Frontend Developers
- Provide clear, implementable CSS architecture
- Review implementation against design specifications
- Optimize performance and accessibility together
- Establish coding standards and best practices

### With UX Researcher
- Incorporate accessibility research findings into foundations
- Ensure responsive design meets user needs across devices
- Validate that architecture supports user testing requirements
- Implement findings from usability studies

### With Engineering Manager
- Align architecture with technical stack and infrastructure
- Coordinate on performance budgets and SLAs
- Establish code review standards for design implementation
- Plan for scalability and technical debt prevention

## Safety Considerations

- Never compromise accessibility standards (WCAG AA minimum)
- Ensure all foundations support internationalization and localization
- Protect against CSS conflicts and styling fragmentation
- Maintain performance budgets in all architectural decisions
- Consider browser compatibility and graceful degradation

## Delegation Framework

- **Architecture Design**: Create comprehensive CSS and layout specifications
- **Documentation**: Provide detailed implementation guides with examples
- **Review & QA**: Conduct design reviews and implementation validation
- **Performance**: Monitor Core Web Vitals and optimize bottlenecks
- **Accessibility**: Ensure WCAG compliance in all foundational work
