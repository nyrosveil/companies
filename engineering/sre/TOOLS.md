# Tools

## Core Technologies
- SLOs (Service Level Objectives) and error budgets
- Observability (metrics, logs, traces)
- Golden signals (latency, traffic, errors, saturation)
- Incident response and post-mortems
- Chaos engineering
- Capacity planning and load testing
- Toil reduction and automation
- Reliability automation

## Key Tools
- Monitoring: Prometheus, Grafana, DataDog, New Relic, VictoriaMetrics
- Tracing: Jaeger, OpenTelemetry, Honeycomb, Lightstep
- Logging: ELK stack, Loki, Splunk, Datadog Logs
- Alerting: Alertmanager, PagerDuty, Opsgenie, VictorOps
- Chaos: Chaos Mesh, Gremlin, Chaos Toolkit, K6, Vegeta
- SLO tracking: Sloth, pyrgo, Google SLO framework
- Infrastructure: Kubernetes, Terraform, Ansible
- CI/CD: GitHub Actions, GitLab CI, Spinnaker, ArgoCD

## Integration
- Incident management platforms
- On-call rotation and escalation
- Post-incident review processes
- Capacity planning tools
- Cost monitoring and optimization
- Service mesh (Istio, Linkerd) for observability
- APM (Application Performance Monitoring)
- Synthetic monitoring and uptime checks
