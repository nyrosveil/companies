# TOOLS.md -- Game Designer

## Core Skills

- `para-memory-files` - Memory operations and PARA methodology
- `paperclip` - Paperclip orchestration and agent management
- `paperclip-create-agent` - Hiring and creating new agents
- `internal-comms` - Internal communications and updates

## Design & Documentation

- **Game Design Document (GDD) Templates**: Core gameplay loop, economy balance spreadsheet, player onboarding flow, mechanic specification
- **Design Pivot Framework**: Prioritize design pillars, deconstruct player motivation
- **Systems Thinking Framework**: Focus on feedback loops, economic chains, and emergence
- **Narrative Design Integration**: Translate story into environmental gameplay

## Game Balancing

- **Economy Modeling Tools**: Supply/demand analysis, inflation detection metrics, player archetype targeting
- **Tuning Spreadsheets**: Math-based progression curves, damage falloff formulas, resource flow analysis
- **Monte Carlo Simulation**: For progression curves and edge case identification
- **Balance Playbooks**: Game-specific balancing patterns for different genres (RPG, platformer, etc.)

## Prototyping

- **Paper Prototype Methods**: Sketching workflows, flowcharting mechanics, card-based systems
- **MVP Design Sprint**: 48-hour rapid design prototype for feasibility testing
- **Design Held Interactions**: List of all interactivity possibilities for weak mechanics
- **Core Loop Stress Test**: Isolate and validate the fundamental fun hypothesis

## Playtesting

- **Playtest Protocol**: Define success criteria, separate observation from interpretation
- **Causal Analysis Techniques**: Root cause analysis of player behavior
- **Player Archetype Mapping**: Identify who's enjoying the experience and who's not
- **Feel Measurement**: Quantify emotional response through facial coding or biofeedback

## Integration Recommendations

- Use para-memory-files for all design knowledge management and GDD versioning
- Employ paperclip for task delegation and progress tracking
- Reference HEARTBEAT.md for daily operational rhythm
- Activate the tdd-guide agent before optimizing any key system
- Limit yourself to 3 design pillars per game — more dilutes focus
- Use GDDs, not meetings, for cross-team alignment
- Say "no" to features that don't support your design pillars
- Draft balance numbers with mathematical models, not gut feel
